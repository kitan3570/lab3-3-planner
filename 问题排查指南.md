# 问题排查和修复方案

## 当前问题

在访问 https://lab3-d3gc0uqhg90f39d16-1433230905.tcloudbaseapp.com/ 时，API 调用失败，显示 "网络请求失败: [object Object]"。

## 可能的原因

1. **CloudBase 云函数访问权限问题** - 云函数可能不允许匿名调用
2. **CORS 配置问题** - 虽然已配置，但可能需要调整
3. **CloudBase JS SDK 初始化问题** - 可能需要登录才能调用云函数
4. **数据权限规则问题** - 数据库访问可能受限

## 解决方案

### 方案一：通过 CloudBase 控制台配置权限（推荐）

#### 步骤 1：配置云函数访问权限

1. 访问 https://console.cloud.tencent.com/tcb
2. 选择环境 "lab3-d3gc0uqhg90f39d16"
3. 进入"云开发" -> "云函数"
4. 对每个函数（plan、location、ai-summary）：
   - 点击函数名
   - 进入"权限管理"或"触发器管理"
   - 确保允许匿名调用，或配置为"所有用户可调用"

#### 步骤 2：配置数据库权限规则

1. 进入"云开发" -> "数据库"
2. 选择"plans"集合
3. 点击"权限设置"
4. 设置为"所有用户可读写"（开发测试期间）
5. 对 "locations" 集合重复同样操作

#### 步骤 3：测试云函数调用

在浏览器控制台或使用 Postman 测试：

```javascript
// 在浏览器控制台测试
const cloudbase = window.cloudbase;
if (cloudbase) {
  const app = cloudbase.init({
    env: "lab3-d3gc0uqhg90f39d16"
  });
  
  // 测试调用 plan 函数
  app.callFunction({
    name: "plan",
    data: {
      httpMethod: "GET",
      path: "/plans",
      body: ""
    }
  }).then(result => {
    console.log("测试成功:", result);
  }).catch(error => {
    console.error("测试失败:", error);
  });
}
```

## 快速验证步骤

### 1. 检查 CloudBase JS SDK 是否加载

在浏览器控制台输入：
```javascript
console.log(window.cloudbase);
console.log(window.cloudbase?.SdkVersion);
```

### 2. 检查当前环境 ID

```javascript
console.log(window?.__tcbData__?.envId);
```

### 3. 查看网络请求

打开浏览器开发者工具 -> Network 标签
- 查看是否有云函数调用请求
- 查看请求的详细信息和响应状态
